Forms

Obtaining User Input

Forms are a common way of allowing a user to interact with a web site.

They require front-end or back-end scripting to be useful.

How forms work

  1. A user enters information
  2. A user submits the information
  3. The input values are processed
    • data is processed using a script (front-end)
    • data is sent to a server (back-end)
  4. Output is produced
    • The current HTML page is updated (front-end)
    • A server returns a new page

Form structure

All inputs should be contained within <form> tags.

Form attributes

  • id: identifies the form
  • action: points to a back-end script
  • method: determines whether the script is receiving information (get) or posting information (post) to a server

Input

Created using the <input> tag. Input tags are self-closing.

The type attribute determines what kind of control is to be displayed.

  • Text
    • text
    • password: text is hidden
  • Choice
    • radio: one choice
    • checkbox: multiple choices
    • select: dropdown or multiple-option box
  • Submit
    • submit: submit a form
    • reset: reset a form

Hidden Input

Sometimes it is necessary to collect data from a page which is not user-submitted.

The hidden input attribute type allows us to specify a value which is not seen by the user.

<input type="hidden" name="current-page" value="home" />

HTML5 Input

HTML5 introduced many input types. These allow forms to be more specifc in regards to what is being asked of a user.

  • color
  • date
  • email
  • number
  • range
  • tel
  • url

Form Validation

What good is data if it is incorrect?

HTML5 offers a required attribute which forces a user to follow constructs when entering data.

<input type="email" required="required" />

Example

HTML5 & older browsers

Not all browsers in use fully support HTML5, which could break our web pages for some users.

If it is necessary to support old browsers, using a polyfill, a script which adds features to a browsers, can help.

Modernizr provides support for HTML5 input types (and other HTML5 features). A custom script can be built to support only the features you need.

Organizing Forms

Depending on the information being asked of users, a form can become daunting to fill out. Elements exist to make forms accessible and organized.

  • label
  • fieldset
  • legend

label

The label tag marks up text which can be associated with an input field.

<label>First Name:
  <input type="text" name="first-name" />
</label>

Example

fieldset & legend

It is possible to group related items using the fieldset tag. The legend tag is used to describe the grouping.

<fieldset>
  <legend>Name</legend>
  <label>First:
    <input type="text" name="first-name" />
  </label>
  <label>Last:

Selecting Forms

The DOM has a built-in object which handles forms.

Rather than select a form with with getElementById, querySelector, etc., we can specify the index number of the form in a document.

document.forms[0] // returns the first form
document.forms[1] // returns the second form

Selecting Form Items

The forms object handles each element within a form.

The element can be specified by its index, or the name given to it in HTML.

<form>
  <input type="text" name="firstName" />
</form>

In a script:

document.forms[0].elements[0]; // firstName
document.forms[0].elements.firstName; // also firstName

Form Controls

The form object has a number of properties and objects which simulate how a user might interact with a form.

  • value: the input entered by a user
  • type: the type of an input
  • name: the name of an input
  • disabled: disables the form
  • checked: is a checkbox checked?
