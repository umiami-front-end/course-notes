CSS Selectors

Selectors

Selectors allow us to identify the part of our HTML to be styled.

Other uses

Selectors are not only used for CSS. The conventions may be used in:

  • JavaScript
  • other language libraries
  • for web scraping

Universal Selector

* {
  ...
}
  • matches elements of any type
  • beware of processing overhead
  • these rules are applied to all elements

Type Selector

a {
  ...
}
  • matches an element by name
  • these rules are applied to all a elements

ID Selector

#cats {
  ...
}
  • matches an element with an id attribute with the specified value
  • the selector value and attribute value must be an exact match
  • id attribute values should be unique
  • these rules are applied to the element with an id attribute of "cats"

Class Selector

.kittens {
  ...
}
  • matches all elements with a class attribute with the specified value
  • these rules are applied to all elements with a class attribute of "kittens"

Attribute Selectors

[href] {
  ...
}
  • matches all elements with the given attribute
  • these rules are applied to all elements which contain a href attribute

Attribute Selector Syntax

Attribute selectors can also focus on contents of an attribute value.

  • [attr=value]: exact match
  • [attr^=value]: begins with
  • [attr$=value]: ends with
  • [attr*=value]: contains one occurrance
  • [attr~=value]: exact match among multiple values
    • values must be whitespace separated
  • [attr|=value]: exact match or begins with value after a hyphen

Adding i to any of these (i.e. [attr=value i]) makes the search case-insensitive.

Combining Selectors

  • simple selectors can be mixed-and-matched
  • offers narrow focus on HTML to be styled
  • can reduce reliance on id, class selectors
  • can reduce amount of code used

Combined

p, li {
  ...
}
  • used when two selectors share the same set of declarations
  • these rules would apply to all p and all li elements

Descendant

footer p {
  ...
}
  • separated by a single space ( )
  • matches the second element if it is an ancestor of the first
  • no limit to how deeply the second element can be nested
  • these rules would apply to only p elements inside footer

Child

ol > li {
  ...
}
  • separated by a greater than symbol (>)
  • matches the second element if it is a direct decendent of the first
  • these rules would apply to only li elements which are direct decendants of ol

General Sibling

h2 ~ p {
  ...
}
  • separated by a tilde (~)
  • matches the second element if it follows the first element
    • does not necessarily have to be immediately after the first
  • both must be children of the same parent element (hence, "siblings")
  • these rules would apply to p elements which follow h2

Adjacent Sibling

h2 + p {
  ...
}
  • separated by a plus sign (+)
  • matches the second element only if it immediately follows the first element
  • both must be children of the same parent element
  • these rules would apply to the first p element which follows h2

Pseudo-classes

a:hover {
  ...
}
  • a keyword added to a selector which specifies state.
  • can select an element based on:
    • its relation in the document tree
    • browser history
    • content status
    • user interaction
  • usually prefaced with a colon (:)

Reference: MDN: CSS Pseudo-classes

Example: Document Tree

  • :first-child: first element of its siblings
  • :last-child: last element of its siblings
  • :nth-child(): nth element of its siblings
    • specify with a value, odd, even)
  • :only-child: element without any siblings

Example: Browser History

  • :link: a link not yet visited
  • :visited: a link which has been visited by the user

Example: Content Status

  • :checked: checkbox, radio button or option is selected
  • :invalid: input fails to validate
  • :read-only: input not editable by the user

Example: Interaction States

  • :hover: a user hovers over the element
  • :active: a user "activates" the element
  • :focus: an input element receives focus

Pseudo-Elements

A keyword added to a selector which styles a specific part of the selected element.

  • ::before: places content before an element
  • ::after: places content after an element
  • ::first-letter: impacts the first letter of a block element
  • ::first-line: impacts the first line of a block element
