APIs

What is an API?

API stands for Application Programming Interface.

APIs allow us to interact with a web site, application, database, etc. and create scripts to incorporate those programs in our own web site.

HTML-based APIs

Our web pages already come with a very important API: The Document Object Model.

HTML5 also has built-in APIs, including:

  • geolocation
  • localStorage
  • sessionStorage
  • canvas

There are other built-in, browser-based APIs which are not part of the HTML5 specification.

geolocation

geolocation API allows us to get the latitude and longitude of a user (when granted permission).

First, we must get the user's location. Then we can gain information about the location.

Storage

The localStorage API allows us to store up to 5 MB of data on our user's device.

We can:

  • setItem
  • getItem
  • removeItem
  • clear

Storage is synchronous, meaning the page must be refreshed for the data to be seen.

This is not the most secure method (so you'd never store sensitive information), but browsers do employ a same origin policy, which means only the host which stored the data can access it.

local or session?

The main difference between localStorage and sessionStorage is the latter is window/tab specific and wipes out when the window/tab is closed.

Canvas

The canvas API allows us to draw raster images using JavaScript.

Several graphing libraries rely on this API.

View Example

Canvas or SVG?

Canvas (raster) is new enough that we are still finding uses for it (some good, some bad).

SVG (vector) is also becoming a more popular option for rendering graphics in the browser.

The benefits to using SVG over Canvas: It can be modified using CSS or JavaScript and is responsive.

At this point: Use what works, recognize options exist, and keep an eye on the industry.

Other APIs

Many sites offer access to their databases, web applications through an API.

Programmable Web provides links to thousands of sites offering API access.

Many sites will also include a "developer" section if they offer API access.

Using APIs

No two web APIs are alike. The way API data is structured can vary.

Most APIs come in one of two popular formats: XML or JSON.

Three steps to getting good at using APIs:

  • Read documentation.
  • Practice parsing data types and structures
  • READ. DOCUMENTATION.

Data Formats: XML

Extensible Markup Language (XML) is similar to HTML, but used to describe structured data. An example of XML is an RSS feed.

It requires a lot of processing, though.

Data Formats: JSON

JavaScript Object Notation (JSON) uses the same format as object literal notation in JavaScript.

{
  "name": "Flynn",
  "age": 4,
  "type": "Tabby",
  "favoriteFood": ["Chicken", "Fish", "Pizza"]
}

Very widely used among other web sites which provide data, but requires processing.

We can also create our own JSON data from JavaScript Objects.

Accessing Web APIs

Most sites will require registration in order to use an API.

Doing so will provide you with an API key, a unique set of characters tied to your account.

The API key allows a site to know how often you (or your application) is accessing their API.

Protect your keys

Most web applications are built using back-end scripts, which are not easily accessible to a user. Most API-driven sites will implement a key on the back-end.

Because JavaScript is front-facing, implementing a key on the front-end means anyone can gain and use your key.

When possible, have your key be processed on the back-end.

Never leave your key accessible via GitHub.

If you must expose the key, keep track of the usage.
