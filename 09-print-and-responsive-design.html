Adding a Print Style Sheet

A print style sheet can be added to an HTML page using <link> or included in the external style sheet using @media print.

HTML:

<link href="print.css" media="print"
  rel="stylesheet" />

CSS:

@media print {
 /* css blocks */
}

It is important to identify the styles are for print, otherwise they may overwrite your screen styles.

Links

Links are unique to the web. Are these necessary in print?

The href attribute is the equivalent of providing a source. Think of it as the digital equivalent of a footnote.

Displaying Links in Print

By combining attribute selectors which match external links, the :after pseudo-element and content property, the value of href can be inserted after the text.

a[href^=http]:after {
  content: " (" attr(href) ") ";
}

Responsive Design

Responsive design assures users receive the same content, regardless of device. It consists of:

  • CSS media queries
  • flexible grid
  • flexible images

The Mobile Viewport

Mobile devices render a (non-mobile) web page with a width of 980 pixels â€” as if it were on a desktop or laptop â€” then shrink it to fit the device's screen dimensions.

This generally makes standard web pages difficult to use.

Defining the Viewport

The meta tag accepts an attribute, viewport*. This allows us to identify a device's screen dimensions.

<meta name="viewport"
  content="width=device-width; initial-scale=1;" />

* This is not actually a W3C standard! W3C has proposed a CSS-based @viewport statement, but browser support is poor.

A Note About maximum-scale

Some web tutorials suggest including maximum-scale=1 in the viewport attribute. Do not include this value.

It hinders the ability for a user to pinch zoom your web site, making it less accessible.

Adding Mobile Styles

When including mobile styles, they can be in a separate style sheet or at the bottom of the existing style sheet. Identify these as "screen" styles.

Add mobile styles after web styles. Take advantage of inheritance.

HTML:

<link href="mobile.css" media="screen"
  rel="stylesheet" />

CSS:

@media (condition) {
  /* css blocks */
}

@media queries

@media queries apply styles targeting specific conditions.

The max-width property is often used for mobile design.

@media (max-width: 640px) {
  /* css blocks */
}

Starting (Break)points

At what point does a phone design look bad on a tablet? Or a laptop design look bad on a desktop?

Another approach is to start with a breakpoint based on a common size for a device and adjust accordingly.

  • Phone (portrait): 400 pixels
  • Tablet (portrait) / Phone (landscape): 768 pixels
  • Tablet (landscape) / Laptop / Desktop: 960 pixels
  • Large Screen: 1280 pixels

Consider adjusting these to relative units.
