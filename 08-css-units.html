CSS Units

When it comes to defining measured values in CSS, there are several options designers can rely on. These can be grouped into two categories:

Absolute units have real-world equivalents.

Relative units are based on the size of something else.

Absolute Units

  • inches (in)
  • centimeters (cm)
  • millimeters (mm)
  • 1/4 millimeter (q)
  • points (pt)
    • 1/72 inch
  • picas (pc)
    • 12 points or 1/6 inch

Using Absolute Units

Absolute units are for print and would only be used in a print stylesheet. In screen (web) design, these are not used.

(So, rather than use a tool like Adobe InDesign to lay out a document, it's technically possible to do it using HTML and CSS!)

Relative Units

  • ems (em)
  • root ems (rem)
  • viewport width (vw)
  • viewport height (vh)
  • viewport minimum (vmin, vm)
  • x-height (ex)
  • zero-width (ch)

What are pixels?

A little bit of both.

The CSS3 specification defines a pixel as 96 pixels per inch, so this qualifies px as an absolute unit. However, this only defines a reference.

Screen resolutions are not always consistent with this reference and vary greatly depending on the device. (See: screensiz.es)

Pixels

Pros:

  • allows for very precise control of design
  • what you see is what you get (WYSIWYG)

Cons:

  • poor choice for fluid, responsive design
  • not ideal for resizing text, designs

Syntax:

property-name: 30px

And percentages?

Percentages (%) are not a formal unit, but are a common way of specifying measurements in CSS.

The "unit" is predominantly used for layout and are relative to the size of the containing element. This assures page layouts remain proportional.

Percentage

Pros:

  • Most HTML default values are percentage based
    • h1 is 200% of the default font size
    • block elements take up 100% of the parent element's width
  • great for fluid, responsive design layouts

Cons:

  • requires calculation based on parent font size
  • can be difficult to determine pixel equivalent without tools
  • very precise calculations can disrupt layout

Syntax

property-name: 90%

ems

  • origin in traditional typography
  • based on the width of a capital M
    • if the font size is 16px, an em is 16px
  • relative to current font size
  • no relation to <em>

ems (continued)

Pros:

  • great for fluid, responsive design fonts, layouts, whitespace
  • once a base font is set, everything can be relative (to the parent)

Cons:

  • requires calculation based on parent font size
  • very precise calculations can disrupt layout
  • inherits a parent's property
    • more to keep track of

Syntax

property-name: 2em;
/* 32px, if base font is 16px */

Root ems

  • the same as an em, but based on root font size (not parent)
  • the "root" is the element node html
    • a browser sets the default size for html at 16px
    • does not work in IE8* or earlier

    * support for IE8 ended in 2016

Root ems (continued)

Pros:

  • great for fluid, responsive design fonts, layouts, whitespace
  • a little more predictable than percentage or ems

Cons:

  • requires calculation
  • can still result in precise, layout-disrupting calculations
  • older browsers (IE 8 or earlier) lack support
    • but Microsoft dropped support in 2016

Syntax

property-name: 1.25rem;
/* 20px if base font is 16px */

Viewport width & height

  • units relative to the browser viewport
  • equal to 1/100th of the viewport width/height
  • useful for keeping elements the full width/height of a viewport

Browser Defaults

Modern browsers have a default font size of 16 pixels. But...

What if a user adjusts these settings?

What if a browser maker changes the default value?*

Using relative units, measurements will still be proportional to the newly defined font size.

* Netscape, Internet Explorer used to have default sizes of 14px.

Converting px to %

Formula:

(size in pixels / parent size in pixels) * 100 = percentage

Examples:

(12px / 16px) * 100 = 75%
(36px / 16px) * 100 = 225%
(48px / 16px) * 100 = 300%

When using percentage, the parent size is based on the immediate parent, the root.

If you are calculating the font size for an element whose parent (or ancestor) also has a font size specified, you must account for it in your calculations.

Converting px to em

Formula:

size in pixels / parent size in pixels = ems

Examples:

12px / 16px = .75em
36px / 16px = 2.25em
48px / 16px = 3em

When using ems, the parent size is based on the immediate parent, not the root.

If you are calculating the font size for an element whose parent (or ancestor) also has a font size specified, you must account for it in your design.

Converting px to rem

Formula:

size in pixels / root font size in pixels = root ems

Examples:

12px / 16px = .75rem
36px / 16px = 2.25rem
48px / 16px = 3rem

When using root ems, the parent size is based on the font-size property set in the css block with html as its selector.

When do I use...?

There is no set standard for using a unit in a specific situation, every time. Some designers pick one and use it throughout. Many use a mix. Some good use cases:

Use percentages and viewport width/height for dimensions related to page layouts.

Use rems and ems for typography, whitespace dimensions. These are sometimes used for layouts, too.

Use pixels for borders.

Tips

When you start a design, using pixels will help you see precisely how your designs are rendered. Convert fixed units to fluid ones in a later iteration of your design.

Once you become familiar with using relative units, try using them over pixels in specific situations (i.e. fonts, margin/padding, etc.).

When making conversions, be sure you leave a comment in your CSS as a reminder of your calculation.

property-name: 1.875em; /* 30px / 16px */
