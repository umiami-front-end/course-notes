JavaScript Syntax

Syntax

A set of rules which define how a computer language should be written.

Comments

Single-line comments are preceded by a double slash.

// Comments may appear at the start of a line or...
var color = "blue";  // ...after a piece of code.

Multi-line comments are enclosed with slashes and stars (like in CSS).

/* This comment
   appears on
   three lines. */

Case Sensitivity

JavaScript is case sensitive.

myCat is not the same as mycat or MYCAT.

Semicolon

A semicolon (;) is used to end each statement.

x = 3 + 2;

Omission of a semicolon may result in an error.

Quotes

Quotes (single or double) distinguish strings from variables, functions, keywords and reserved words.

'Hello!' is a string.

"Hello, world!" is also a string.

helloWorld is not a string. It may be a variable or function.

Parentheses

Parentheses enclose a parameter to a function; an argument to a method; or are used to change the order of operations in arithmetic expressions.

function sayHello(name) { ... }

document.write("Hello!");

var solution = ((4 + 76) / 3);

Curly Braces

Curly braces ({ }) group statements into compound statements or "blocks" such as functions or dependent statements.

var x = 2;
if (x == 2) {
  document.write("the variable x holds");
  document.write(x);
};

Period

A period or "dot" (and word following) is used to denote a property or method of an object.

document.write("hello");
// the method "write" is of the object "document"

Reserved words

All programming languages have reserved words which describe built-in functions, objects, properties, methods, etc.

For example: break, else, for, if and while are among the reserved words in JavaScript. These words cannot be used to declare variables, functions, methods, etc.

Once you are "fluent" in a programming language, these words become familiar. Until then, you may need to refer to resource for a list of reserved words.

See: Mozilla Developers Network: JavaScript Reserved Words

Variables & Constants

  • containers which hold data
  • have no value until one is assigned
  • must have a unique name
  • can be referenced to retrieve its value

Declaring Variables

When we create a variable, preface the name with var (or let).

Once a variable exists, we do not need to include var (or let) in future references.

Variables Syntax

  • must start with a letter, dollar sign ($) or underscore (_)
  • may not contain a dash (-) or period (.)
  • are case sensitive
  • should be descriptive
  • should be written in camelCase* (var lowerCapsCaps;)
  • may not contain keywords or reserved words

* underscores are acceptable, but camelCase is preferred JavaScript style

Constants

  • a variable whose value will not change
  • identified in human-readable form by using ALLCAPS
  • returns an error if there is an attempt to change its value
const PI = 3.14;
