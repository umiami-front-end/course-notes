JavaScript 101

JavaScript

  • a high-level, interpreted language
  • the secret sauce of web interactivity
  • introduced in Netscape Navigator 2.0 (1995)
  • commonly used in browsers for client-side scripts
  • has gained popularity for server-side uses (i.e. node.js)

JavaScript is not Java!

JavaScript and Java are not the same. They have different syntaxes and uses.

JavaScript is an interpreted language. Java is compiled.

Please don't say we are learning Java ... this is a teeny pet peeve. >:}

Standard JavaScript

ECMA (European Computer Manufacturers Association), through assistance from Netscape, developed a standard version of JavaScript in 1997.

The standardized version is ECMAScript, but everyone still calls it JavaScript.

The language is continually evolving. New features are added yearly.

ECMAScript 6 (2015) is supported by 98 percent of current browsers. (See: ECMAScript compatibility table)

Considerations

JavaScript is standardized and widely adopted, but some users:

  • may turn off JavaScript functionality
  • surf with older browsers which may lack ES5/6 support
  • lack bandwidth to download large scripts, libraries
  • lack computing power to handle client-side processing

JavaScript Console

Modern browsers include a command-line interface which lets us work step-by-step with JavaScript.

Chrome: View → Developer → JavaScript Console

Firefox: Tools → Web Developer → Web Console

REPL

The JavaScript Console is the example of a REPL:

  • Reads your command
  • Evaluates the statement or expression
  • Prints a result
  • Loops ... repeats the previous steps!

Using the Console

  • tinker with very simple statements
  • interact with window, document object models
  • be alerted to errors when code is incorrect
  • debugging
  • isolate parts of a script loaded into the browser

Tips

If you are unsure about how an aspect of JavaScript works, try entering an example in the console.

If the example requires multiple lines of code, type shift-return at the end of each line to keep the code from executing. The cursor will move to the next line so you can continue typing your command(s).

Testing Big Scripts

Sometimes it is easier to write code in a file than in the console. Include the script in a basic HTML page, then load the HTML page into the browser.

Use this method when scripts become too cumbersome for the browser's REPL.
