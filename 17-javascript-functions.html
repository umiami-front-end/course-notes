Functions

What is a function?

A function is a reusable grouping of statements which perform a specific task.

Function Expression

function functionName() {
  // code goes here
}
  • function keyword indicate the start
  • a unique name followed by parentheses
  • curly braces wrap the code inside

Arrow Function

() => {
  // code goes here
}
  • introduced in ES6
  • a more concise function expression

Anonymous Function

var variable = function() {
  // code goes here
}

// or...

function() {
  // code goes here
}
  • no formal name is used
  • may be stored in a variable or executed immediately
  • often used as arguments for methods

Calling a Function

After a function has been defined, we call the function to execute the code within.

function collegeCheer() {
  console.log("Go Canes!");
}

// more code ...

collegeCheer();

Parameters & Arguments

Parameters are defined in the function expression:

function myNewFunction(parameterName) {
  // code ...
}

Arguments the values passed to a function.

myNewFunction(argumentValue);

Returning a Value

After a function has performed its role, we often need to return a value.

We use a return statement at the end of a function to do so.

Function with Parameters

function addition(value1, value2) {
  return value1 + value2;
}

The terms used inside the parentheses match the terms inside the function.

The value our function returns is the result of adding value1 and value2.

Calling a Function (with Arguments)

addition(3, 4);

Or...

var number1 = 3;
var number2 = 4;

addition(number1, number2);

Local Variables & Scope

A local variable is a variable contained within a function. This variable cannot be accessed from outside of the function.

Local variables are said to have local scope.

function hello() {
  var message = "Hello!";
  return message;
}

document.write(message);  // message is not defined

Global Variables & Scope

A global variable is a variable declared outside of a function. It is accessible anywhere in the script.

Global variables are described as having global scope.

var message = "Hello!";
function hello() {
  return message;
}

document.write(message);
hello(message);

var vs. let

let is more strict in its use, primarily relating to scope. Variables declared with let:

  • scoped to the immediate enclosing block
  • can only be used after defined
  • do not create a property on the global object
  • cannot be re-declared

this

this is a keyword which exists inside functions. It limits our methods to the properties of the object calling the function.

If you have programmed using an object-oriented language like Java or Ruby, this is like self.

Function Expression

When we use this inside a function, it will use the value specified within an object.

var rectangle = {height: 20, width: 30};
var area = function() {
  return this.width * this.height;
}

rectangle.calculateArea = area;
rectangle.calculateArea();

this with Global Variables

We can also use this in a function which refers to a global variable.

var color = "red";
var square = {color: "blue"};

function getColor() {
  console.log(this.color);
}

getColor();

Notice how getColor(); returns the value of the global variable color, not the color property inside square.

this & Global Scope

If no global variables are defined, the this keyword used within a function will look for the property of the window.

function windowHeight() {
  console.log(this.innerHeight);
}

windowHeight();

Object Method

Rather taking an extra step to add a function to an object, we can build the function into our object from the start.

var rectangle = {
  height: 20,
  width: 30,
  calculateArea: function() {
    return this.height * this.width;
  }
};

rectangle.calculateArea();
