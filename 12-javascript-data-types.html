JavaScript Data Types

Primitive Data Types

The most basic 'building blocks' which can be used in programming.

  • Boolean
  • number
  • string
  • null
  • undefined

Booleans

There are only two Boolean values:

  • true
  • false

Booleans are common results of conditional tests. (i.e. "If true, do X; if false, do Y")

Numbers

If made it this far in academia, you should be familiar with these. üòÅ

  • integers (1, 2, 3)
  • floating-point numbers (a.k.a. "floats") (1.0, 2.0, 3.0)
  • special numbers (Infinity, -Infinity, NaN)

Numbers are used in calculations, comparisons, counting, date/time.

Strings

Strings are a combination of characters which represent text or code.

Strings which represent text are enclosed in single or double quotes. (i.e. "string")

Strings may be concatenated (linked together).

"Hello, " + "world!"; // "Hello, world!"

null

null is an object which has no value on purpose. It is not false or empty.

var dev = null;  // null
dev = false;     // false
dev = "";        // ""

undefined

Something is undefined when it exists, but contains no value.

var cats;
cats;  // undefined

NaN

NaN is a property which represents a value being "Not a Number". It is returned when Math or attempts to parse a number fail.

It is a wonderful example of JavaScript's weirdness.

Working with Data Types

Data by itself is useless. We must act on the data to process it.

JavaScript includes the following types of operators:

  • assignment
  • arithmetic
  • string
  • comparison
  • logical

Assignment

The assignment operator (=) assigns a value to a variable.

var room = 2054;
room;  // 2054

Arithmetic

  • addition (+)
  • subtraction (-)
  • multiplication (*)
  • division (/)
  • modulo (%)
  • increment (++)
  • decrement (--)

Order of Operation

Programming has mathematical roots, so calculations follow Algebra's order of operation (multiplication, division before addition, subtraction).

If you need to change this, enclose the calculation in parentheses ( ( ) ).

20 + 30 * 10;     // 320
(20 + 30) * 10;   // 500

Arithmetic & Assignment

Arithmetic and assignment operators can be combined, creating a shortcut for modifying values.

  • add and assign (+=)
  • subtract and assign (-=)
  • multiply and assign (*=)
  • divide and assign (/=)
  • modulus and assign (%=)

These, too, are often used in loops when each step is greater/less than 1.

Calculate & Assign

The long way:

var x = 2;
x = x * 2;
x;

The short way:

var x = 2;
x *= 2;
x;

These work the same way; use is a matter of preference.

String

The concatenation (+) operator to combines strings.

"Hello " + "world!";  // Hello, world!

Strings & Numbers

JavaScript is an untyped language, which means we never have to explicitly say what our data is. JavaScript knows 3 is a number and "3" is a string.

Let's prove this:

typeof 3;    // number!
typeof "3";  // string!

Numbers as Strings

If we enclose numbers in quotes, JavaScript will treat them as strings.

"1" + "2";  // "12"

If we concatenate a number with a string, the number becomes part of the string.

2018 + " Miami Hurricanes";  // "2018 Miami Hurricanes"

Try these!

"10" - "5";  // What?!
"10" * "5";  // WHAT?!?!

(Just one example why JavaScript is regarded as quirky.)

Strings Are Not Numbers

If we 'calculate' strings, the result is NaN or "Not a Number."

"ten" - "five";  // NaN

Comparison

Comparison operators are used when our code makes decisions or loops.

When one value is compared to another, the return value is true or false.

  • greater than (>)
  • greater than or equal to (>=)
  • less than (<)
  • less than or equal to (<=)
  • equal to (==)
  • not equal to (!=)
  • strict equal to (===)
  • strict not equal to (!==)

Strict or Not?

Using strict equal to (===) or strict not equal to (!==) compares the value and data type.

"10" === 10;    // false
"10" !== "10";  // false
10 === 10;      // true

Logical

Logical operators are used when our code makes decisions or loops.

Logical operators compare expressions and return true or false values.

  • and (&&)
  • or (||)
  • not (!)

and

Both expressions/values must evaluate to true to return a true value.

true && true;    // true
true && false;   // false
false && true;   // false
false && false;  // false

or

One expression/value must evaluate to true to return a true value.

true || true;    // true
true || false;   // true
false || true;   // true
false || false;  // false

not

The opposite value is returned.

!true;   // false
!false;  // true
