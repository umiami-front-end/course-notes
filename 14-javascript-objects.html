JavaScript Objects

Objects

A collection of...

  • properties (descriptions)
  • methods (actions)

How can we describe a cat?

Cat: Descriptions and Actions

We might describe a cat by its:

  • coat
  • number of paws
  • colors
  • personality (shy? friendly? bites?)

A cat also acts in the following ways:

  • says "Meow"
  • purrs
  • pounces
  • sleeps

Quorra (as a JavaScript Object)

var quorra = {

  // properties
  species: "cat",
  paws: 4,
  coat: "soft",
  colors: ["brown", "gray", "black", "white"],
  shy: true,
  friendly: true,
  bites: false,

  // a method
  speak: function() {
    return 'Meow!';
  }
}

Built-In Objects

Each time a data type is created, JavaScript automatically associates it with specific properties and methods.

We have already seen:

  • String
  • Number
  • Boolean
  • Array

See: MDN: Standard built-in objects

Object Literal

  • defined by key-value pairs
    • keys are like variables
    • keys end with a colon (:)
    • values end with a comma
    • values may be any data type
  • pairs are placed within curly brackets ({})
  • pairs may be inline or on individual lines
  • separated by commas
  • last pair does not require a comma
var car = {
  make: "Chevrolet",
  model: "Camaro",
  year: 2016
}

Object Constructor

A new object is created using new Object();.

Key-value pairs are added using dot notation on the variable (to append the property/method name). The value is added using the assignment operator.

var car = new Object();
car.make = "Chevrolet";
car.model = "Camaro";
car.year = 2016;

Accessing Object Items

An object's properties or methods may using either bracket or dot notation.

While you learn, you may want to use bracket notation since it is also used when accessing array values. (You cannot access an array value using dot notation.)

Bracket Notation

Bracket notation uses the key to reference the value. The key should be enclosed in double quotes.

car["make"];
car["model"];
car["year"];

Dot Notation

Dot notation uses the key after a period to reference the value. The key should not be quoted.

car.make;
car.model;
car.year;

Changing Values

To change the value of a property/method, simply reassign it a new value.

car["model"] = "Corvette";

Deleting Keys

If you want to delete a property/method:

delete car.model;
