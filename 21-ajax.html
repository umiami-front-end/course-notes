Ajax

What is Ajax?

"Asynchronous JavaScript and XML"

  • a processing model which does not interfere with user
  • loads data without refreshing an entire page

Understanding Web Servers

The web has its own protocol: Hypertext Transfer Protocol (HTTP).

  • facilitates communication between a client and server
  • a client (web browser) requests a file from a web server
  • the web server acknowledges the request by sending response
  • if the response is "OK!", the requested file is sent

Requests

The most common requests a client can make to a server are GET and POST.

  • GET retrieves data
    • most common request when browsing the web
  • POST sends data back to a server to be saved
    • post to a message board, update a database, submit a form

HTTP Status Codes

When a client makes a request, it will respond with a status code.

  • 1xx: Informational
  • 2xx: Success
    • 200: OK! (file transfer begins)
  • 3xx: Redirection
    • 301: Moved Permanently
    • 304: Not modified
  • 4xx: Client Error
    • 404: File Not Found
  • 5xx: Server Error
    • 500: Internal Server Error

How Ajax Works

When a browser is rendering HTML, CSS and JavaScript, it may come across an Ajax request object (XMLHttpRequest).

As the browser builds a page, the objects hits the server for specific data. The server sends a response, which fires the onload event in the browser.

Think of Ajax as a runner, assistant focused on a very specific task as a major event is ongoing.
